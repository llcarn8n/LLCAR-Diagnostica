# План: Умные карточки базы знаний

> Задача: сделать результаты поиска УМНЕЕ и ЧИТАБЕЛЬНЕЕ, не меняя архитектуру.
> Дата: 2026-03-05

---

## Проблема

11 000 статей выглядят одинаково. Нет разницы между "ОСТАНОВИСЬ, ОПАСНО" и "история модели".
Не видно откуда информация, для какого пробега, когда актуально.

## Три шага

### Шаг 1. Паспорт статьи — `build_situation_tags.py`

Автоматическая разметка каждой из 11 398 статей:

| Поле | Значения | Как вычисляется |
|------|----------|-----------------|
| `urgency` (1-5) | 5=пожар/взрыв, 4=HV батарея/тормоза, 3=DTC, 2=обычная, 1=справка | Ключевые слова + layer + has_warnings |
| `situation_type` | emergency / maintenance / troubleshooting / learning / specification | has_warnings + has_procedures + content_type + ключевые слова |
| `trust_level` (1-5) | 5=офиц.мануал, 4=мануал(перевод), 3=СМИ, 2=отзыв, 1=неверифиц. | Маппинг из source |
| `season` | winter / summer / all | Ключевые слова в контенте (зим/мороз/лет/перегрев) |
| `events` | warning_light / noise / vibration / performance / maintenance / smell / leak / no_start | Ключевые слова в контенте |
| `mileage_ranges` | 0-10k / 10-30k / 30-60k / 60k+ | Тип операции + layer |

Скрипт: чистый SQL + regex, ~30 секунд, без нейросетей.
Выход: таблица `situation_tags` в kb.db + JSON-файл.

### Шаг 2. Умные карточки в UI

**Экстренная карточка** (urgency >= 4):
- Красная рамка 2px #EF5350
- Красноватый фон rgba(239,83,80,0.08)
- Всегда наверху списка результатов

**Индикатор доверия** (все карточки):
- 5 точек: ●●●●○ = мануал(перевод), ●●○○○ = отзыв владельца
- Рядом текст: "Офиц. мануал" / "Веб-источник"

**Индикатор перевода** (если source_language !== user_language):
- Жёлтая метка: `ZH → RU Автоперевод`
- Данные уже есть в ответе API

**Бейдж типа** (из situation_type):
- Красный "Внимание" для emergency
- Синий "Процедура" для maintenance
- Серый "Справка" для learning

### Шаг 3. Блок "Сейчас важно"

Над результатами поиска — 2-3 контекстных карточки:
- Зима (окт-мар) → запрос "подготовка зима" с фильтром season:winter
- Лето (апр-сен) → запрос "охлаждение лето кондиционер"
- Если есть DTC в localStorage → карточка с DTC
- Если пробег → напоминание о ТО
