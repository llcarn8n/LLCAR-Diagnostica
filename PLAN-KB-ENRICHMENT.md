## Проблема

KB на **93% состоит из OCR-мануалов** (11,024 чанков). Она отлично покрывает "как устроено", но почти не покрывает **"как пользоваться"** и **"что делать когда сломалось"**. Web-контент — всего 2.5% (301 чанк), и большая часть — новостной шум.

Скраперы собирают всё подряд без фильтрации по релевантности: новости о продажах, пресс-релизы для инвесторов, маркетинговые описания. Ценный контент (отзывы владельцев, пошаговые ремонты, реальные проблемы) составляет <30% собранного.

### Ключевые находки по Drive2.ru

Drive2 блокирует обычные HTTP-запросы. Решение (по приоритету):
1. **curl_cffi** с `impersonate="chrome"` — быстрый, без браузера
2. **stealth_fetch** (patchright) — уже есть в base_scraper.py
3. **SeleniumBase UC Mode** — гарантированно работает, но медленно

На GitHub **нет готовых скраперов drive2.ru**. Скрапер обновлён: использует `_smart_fetch()` с тремя fallback-методами.

## Улучшения скрапперов

### 1. Фильтр релевантности (в base_scraper.py)

Добавить scoring по ключевым словам диагностики. Статьи с score < 0.2 — отбрасывать.

```python
DIAGNOSTIC_KEYWORDS = {
    'ru': ['ремонт', 'обслуживание', 'проблема', 'поломка', 'ошибка', 'код', 'замена',
           'ТО', 'подвеска', 'масло', 'фильтр', 'тормоз', 'аккумулятор', 'батарея',
           'неисправность', 'диагностика', 'сервис', 'пробег', 'расход', 'зарядка',
           'пневмо', 'скрип', 'вибрация', 'перегрев', 'ошибка', 'индикатор',
           'обновление', 'OTA', 'запас хода', 'колодки', 'антифриз'],
    'en': ['repair', 'maintenance', 'problem', 'issue', 'fault', 'error', 'code',
           'replace', 'service', 'suspension', 'oil', 'filter', 'brake', 'battery',
           'troubleshoot', 'diagnostic', 'mileage', 'recall', 'charging', 'range',
           'NVH', 'noise', 'vibration', 'overheat', 'OTA', 'update'],
}

def relevance_score(content: str, lang: str = 'ru') -> float:
    keywords = DIAGNOSTIC_KEYWORDS.get(lang, DIAGNOSTIC_KEYWORDS['en'])
    text = content.lower()
    hits = sum(1 for kw in keywords if kw.lower() in text)
    return min(hits / 5.0, 1.0)  # 5+ совпадений = 1.0
```

### 2. Извлечение DTC кодов из текста

```python
import re
DTC_PATTERN = re.compile(r'\b[PCBU]\d{4,5}\b')

def extract_dtc_codes(content: str) -> list[str]:
    return sorted(set(DTC_PATTERN.findall(content.upper())))
```

### 3. Классификация при импорте

Автоматически назначать `content_type` вместо `web_scraped`:
- `troubleshooting` — если DTC коды или слова "проблема/поломка/ошибка"
- `maintenance` — если "ТО/обслуживание/замена масла/фильтр"
- `owner_review` — если с drom_reviews или drive2
- `specs` — если >50% контента — табличные данные ТТХ
- `guide` — если "инструкция/как/пошагово"
- `news` — для остального (самый низкий приоритет)

### 4. Структурированный парсинг Drom отзывов

Извлекать из HTML:
- Рейтинг (звезды 1-5)
- Блоки "Достоинства" / "Недостатки"
- Пробег на момент отзыва
- Год выпуска / комплектация
- Категория проблемы (из "5 копеек")

## Ключевые решения

1. **Drive2 — приоритет #1**: самый ценный источник для русскоязычных владельцев Li Auto. 100+ бортжурналов с реальными ремонтами, ТО, проблемами.
2. **Фильтрация > объём**: лучше 100 качественных чанков чем 1000 шумных. Relevance score ≥ 0.2 отсекает ~60% мусора.
3. **Структурированные данные**: извлекать DTC коды, пробег, категории из текста — не просто сохранять raw text.
4. **Ручной контент**: для P0-пробелов (ежедневная эксплуатация, зима) проще написать 20 качественных гайдов чем скрапить 200 форумных постов.
5. **Telegram**: огромный потенциал (40K подписчиков), но требует тщательной фильтрации — 90% сообщений это чат, не контент.
